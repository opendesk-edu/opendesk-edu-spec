## Current Release: OpenSpec v1.0

The current release establishes the foundational specification framework
for the openDesk Edu ecosystem, covering all 24 services, platform
infrastructure, and integration patterns.

## Near-Term (2026 Q3–Q4)

| Milestone | Description | Status |
|-----------|-------------|--------|
| **Automated spec validation** | CI pipeline validating spec completeness, broken links, and SPDX compliance | Planned |
| **Service test coverage** | Expand E2E test coverage mapped to spec requirements | In progress |
| **i18n content** | Add German, French, and Chinese translations for core specs | Planned |
| **PDF export** | Generate downloadable PDF bundles per service category | Planned |

## Medium-Term (2027 H1)

| Milestone | Description |
|-----------|-------------|
| **API contract testing** | Automated WOPI, LTI, and OIDC contract validation |
| **SLO monitoring integration** | Link spec SLIs to live Prometheus metrics |
| **Compliance automation** | Automated GDPR/BSI compliance checklist generation from specs |
| **Plugin architecture** | Allow third-party service specs as community contributions |

## Long-Term Vision

- **Living specification**: Auto-updating deployment manifests from spec changes
- **Community governance**: Establish a formal open-source foundation
- **Cross-ecosystem standards**: Drive adoption of OpenSpec methodology in
  other digital sovereignty initiatives (OpenDesk, Phoenix,等)

## How to Influence the Roadmap

- Open a [GitHub issue](https://github.com/opendesk-edu/opendesk-edu/issues)
  with the `roadmap` label
- Submit a pull request updating this page
- Discuss in the community channels
