## Project Stewardship

The openDesk Edu OpenSpec is stewarded by **HRZ Marburg** as part of the
openDesk Edu ecosystem. Governance decisions follow the principles of
open collaboration, meritocratic contribution, and transparent decision-making.

## Roles

### Maintainers

Maintainers are responsible for:
- Reviewing and merging specification changes
- Ensuring consistency across the OpenSpec
- Resolving disputes about spec content
- Managing releases and versioning

### Contributors

Contributors propose changes via pull requests. Anyone may contribute
by submitting improvements, corrections, or new specifications.

### Reviewers

Reviewers provide technical feedback on spec changes. Reviewers are
subject-matter experts for specific domains (auth, platform, services).

## Decision-Making

- **Lazy consensus**: Changes are accepted unless objected to within a
  reasonable review period (typically 7 days).
- **Blocking objections**: Valid technical concerns MUST be resolved
  before a change is merged. Preference concerns (style, formatting)
  are non-blocking.
- **Escalation**: Disputes that cannot be resolved among contributors
  are escalated to the maintainers for a final decision.

## Contribution Process

1. Fork the repository and create a feature branch.
2. Make changes following the OpenSpec methodology.
3. Submit a pull request with a clear description of the change.
4. Address review feedback within 14 days or the PR may be closed.
5. Once approved, a maintainer merges the change.

## Code of Conduct

All contributors are expected to follow the [Contributor Covenant](https://www.contributor-covenant.org/)
Code of Conduct. Harassment, discrimination, or toxic behavior will not be tolerated.
