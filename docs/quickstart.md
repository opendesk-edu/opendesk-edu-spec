# Quickstart Guide

Welcome to the openDesk Edu OpenSpec! This guide will help you navigate the specification framework quickly.

## What is the OpenSpec?

The **OpenSpec** is a comprehensive specification framework that describes every aspect of the openDesk Edu ecosystem using a standardized format. Think of it as a **blueprint** for building, deploying, and operating the platform.

## 5-Minute Tour

### 1. Explore the Architecture (1 minute)

Start with the [Introduction](/docs/intro) to understand the overall structure.

### 2. Understand the Methodology (2 minutes)

Read the [Methodology](/docs/methodology) page to learn how specifications are structured.

### 3. Browse Services (1 minute)

Visit the **Services** section to explore the 25 integrated services. Each service has:
- Purpose and scope
- Functional requirements
- Dependencies
- SLOs and DR procedures

### 4. Check Integrations (1 minute)

The **Integration Specifications** show how services work together:
- API contracts
- Cross-service workflows
- Authentication flows

## Common Tasks

### I want to deploy openDesk Edu

1. Read the [Backup](/docs/platform/backup) specification
2. Review the [Security](/docs/platform/security) requirements
3. Check the [Operations](/docs/platform/operations) runbooks
4. Follow the [Quickstart Guide on opendesk-edu.org](https://opendesk-edu.org)

### I want to contribute

1. Read the [Contributing Guide](/docs/community/contributing)
2. Check the [Test Coverage Gaps](/docs/registry/test-coverage-gaps)
3. Open a Pull Request

### I want to understand a specific service

1. Navigate to **Services** in the sidebar
2. Select the service
3. Review its specification
